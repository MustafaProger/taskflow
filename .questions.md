### Что такое npm?

> npm - это менеджре пакетов (библиотек) для JS
- пакет - готовый код, который написал кто-то другой (react, axios, tailwind)

### Что такое npx?

> npx — это инструмент для запуска пакетов (программ) без установки глобально

### За что отвечают файлы config?

> config файлы - это настройки для инструментов (Vite, tailwind, Typescript) в проекте. Им нужно объяснить как работать, поэтому для этого нужны файлы config

### Что такое PostCSS и как он работает?

> PostCSS — это инструмент, который обрабатывает CSS с помощью плагинов.

Простой смысл:
👉 есть CSS → PostCSS прогоняет его через плагины → получается готовый CSS для браузера

Как это работает в проекте:

1. Ты пишешь классы в React:
```jsx
className="bg-black p-4"
```

2. В CSS есть специальные директивы Tailwind:

```css
@import "tailwindcss";
```

3. Ты запускаешь проект (`npm run dev`)

4. Vite находит CSS файл и передаёт его в PostCSS

5. PostCSS смотрит конфиг и видит плагин Tailwind

6. PostCSS вызывает Tailwind

7. Tailwind:
   - смотрит твои файлы (content)
   - находит классы (bg-black, p-4)
   - генерирует обычный CSS

8. Tailwind возвращает готовый CSS обратно в PostCSS

9. PostCSS отдаёт результат Vite

10. Vite отправляет CSS в браузер

👉 Важно:
- PostCSS управляет процессом
- Tailwind — это плагин, который генерирует CSS

👉 В Tailwind v4 директивы @tailwind больше не используются, вместо них используется @import "tailwindcss".

### Что такое autoprefixer и зачем он нужен?

> Autoprefixer — это инструмент (плагин PostCSS), который автоматически добавляет CSS-префиксы для разных браузеров.

Зачем это нужно:
- разные браузеры по-разному поддерживают CSS
- некоторые свойства требуют префиксы (-webkit-, -moz- и т.д.)

Пример:

Ты пишешь:
```css
.btn {
  display: flex;
}
```

Autoprefixer превращает в:
```css
.btn {
  display: -webkit-flex;
  display: flex;
}
```

Как он работает в цепочке:
- PostCSS вызывает Tailwind → получает CSS
- затем вызывает autoprefixer
- autoprefixer добавляет префиксы
- возвращает результат обратно в PostCSS

👉 Важно:
- autoprefixer ничего не генерирует сам
- он только "допиливает" CSS для совместимости браузеров

👉 В Tailwind v4 autoprefixer чаще всего не требуется, так как современные сборщики (например Vite) и сам Tailwind уже покрывают большинство кейсов.

### Что такое layer в Tailwind?

> @layer — это механизм, который управляет порядком применения стилей в Tailwind.

В Tailwind есть 3 слоя:

- base — базовые стили (html, body, reset)
- components — переиспользуемые классы (button, card и т.д.)
- utilities — утилиты Tailwind (p-4, flex, bg-red-500)

Порядок такой:
```
base → components → utilities
```

👉 Главное правило: чем позже применяется слой, тем он сильнее.

Это значит:
- components могут переопределять base
- utilities могут переопределять всё

Зачем это нужно:

Чтобы стили применялись предсказуемо и не конфликтовали между собой.

---

👉 Важно:

- @layer — это только про порядок CSS
- он НЕ создаёт стили сам, а только управляет их приоритетом

---

👉 Отдельно про Tailwind v4:

В Tailwind v4 появилась директива `@theme`, но:

- `@theme` — это НЕ слой
- `@theme` используется для задания design tokens (цвета, шрифты и т.д.)
- он создаёт utility-классы (например: bg-background, text-primary)  [oai_citation:0‡tailwindcss.com](https://tailwindcss.com/docs/theme?utm_source=chatgpt.com)

Пример:

```css
@import "tailwindcss";

@theme {
  --color-background: #fafafa;
}
```

Теперь можно писать:
```html
<div class="bg-background"></div>
```

### В чем разница между `:root` и `@theme`?

На первый взгляд они похожи, потому что и там, и там можно объявлять переменные.
Но роль у них разная.

#### `:root`

`:root` — это обычный CSS-селектор верхнего уровня.
Через него объявляют CSS-переменные, которые потом используются вручную через `var(--...)`.

Пример:

```css
:root {
  --color-brand: #4f46e5;
  --radius-card: 1.5rem;
}
```

Использование:

```css
.card {
  background-color: var(--color-brand);
  border-radius: var(--radius-card);
}
```

👉 То есть `:root` — это просто способ хранить и переиспользовать значения в обычном CSS.

---

#### `@theme`

`@theme` — это специальная директива Tailwind v4.
Через неё задаются theme tokens: цвета, размеры текста, шрифты и другие значения дизайн-системы.

Пример:

```css
@import "tailwindcss";

@theme {
  --color-brand: #4f46e5;
  --color-background: #fafafa;
  --text-default: 16px;
}
```

После этого Tailwind понимает, что:
- `--color-brand` — это цветовой токен
- `--color-background` — это тоже цветовой токен
- `--text-default` — токен размера текста

И на их основе создаёт utility-классы.

Например:

```html
<div class="bg-brand text-brand"></div>
<p class="text-default"></p>
```

👉 То есть `@theme` не просто хранит значение, а ещё сообщает Tailwind, как использовать это значение внутри utility-системы.

---

#### Коротко

- `:root` → обычные CSS-переменные
- `@theme` → theme tokens для Tailwind

Практически:
- если значение нужно только внутри CSS, часто достаточно `:root`
- если значение должно стать частью дизайн-системы и Tailwind-классов, лучше использовать `@theme`

---

### Что делает `@theme` в Tailwind v4?

`@theme` нужен для описания дизайн-токенов проекта.

Он позволяет задавать:
- цвета
- размеры текста
- шрифты
- отступы
- брейкпоинты
- и другие значения темы

Пример:

```css
@theme {
  --color-primary: #111827;
  --color-muted: #f3f4f6;
  --font-display: "Inter", sans-serif;
  --text-body: 16px;
}
```

Что происходит дальше:

1. Ты объявляешь токены в `@theme`
2. Tailwind читает их
3. Tailwind понимает namespace токена
4. Tailwind создаёт utility-классы

Пример логики:
- `--color-primary` → `bg-primary`, `text-primary`, `border-primary`
- `--font-display` → классы для шрифта
- `--text-body` → классы для размера текста

👉 То есть `@theme` связывает значения темы с Tailwind utilities.

### Что такое токен?

Токен — это именованное значение из дизайн-системы.

Проще:

> токен = переменная со смыслом

У токена обычно есть 3 части:
- категория
- имя
- значение

Пример:

```css
--color-primary: #111827;
```

Разбор:
- `color` — категория токена
- `primary` — имя токена
- `#111827` — значение токена

То есть токен отвечает не только на вопрос:
- «какое это значение?»

но и на вопрос:
- «за что оно отвечает в интерфейсе?»

Например:

```css
--color-danger: #ef4444;
--color-foreground: #111827;
--text-body: 16px;
```

Это уже не просто случайные переменные, а осмысленные значения:
- `danger` — цвет ошибки / опасного действия
- `foreground` — основной цвет текста
- `body` — стандартный размер текста

👉 Поэтому токен — это не просто число или цвет.
Это значение с понятной ролью в интерфейсе.

---

### Чем токен отличается от обычной переменной?

Обычная переменная может быть технической:

```css
--temp-value: 17px;
```

Из названия непонятно, зачем она нужна.

А токен обычно описывает роль в системе:

```css
--color-primary: #111827;
--radius-card: 1.5rem;
```

Здесь уже понятно:
- `primary` — основной цвет
- `card` — радиус карточки

👉 Запомнить можно так:

- переменная — это просто способ хранить значение
- токен — это осмысленное значение внутри дизайн-системы

### Что такое @property и как он работает?

#### Проблема

У нас есть кастомное CSS-свойство:

```css
--angle-1: -75deg;
```

По умолчанию браузер воспринимает его как строку:

```
"-75deg"
```

Когда значение меняется:

```css
--angle-1: -100deg;
```

браузер просто делает замену строки.

👉 Он не понимает:
- что это угол
- как считать промежуточные значения

Поэтому анимации (transition) не работают.

---

#### Задача

Нужно сделать так, чтобы значение менялось **плавно**, например:

```
-75deg → -76deg → -77deg → ... → -100deg
```

Для этого браузер должен:
- понять тип значения
- уметь интерполировать (считать промежуточные значения)

---

#### Решение — @property

```css
@property --angle-1 {
  syntax: "<angle>";
  inherits: false;
  initial-value: -75deg;
}
```

Что мы делаем:

- `syntax` — говорим браузеру тип (например `<angle>`)
- `inherits` — управляем наследованием
- `initial-value` — задаём дефолт

👉 Главное:
мы **регистрируем кастомное свойство в движке браузера**

---

#### Как это работает внутри (движок)

Без `@property`:

```
"--angle-1" = строка
```

👉 браузер:
- подставляет значение через `var()`
- не участвует в вычислениях
- не может анимировать

---

С `@property`:

```
--angle-1 = типизированное значение (<angle>)
```

👉 браузер:
- понимает тип (угол)
- извлекает число (-75)
- может интерполировать
- считает промежуточные значения

---

#### Что происходит при анимации

```css
.glass-surface:hover {
  --angle-1: -100deg;
}
```

👉 браузер делает:

```
-75deg → -76deg → -77deg → ... → -100deg
```

И на каждом шаге:
- пересчитывает `conic-gradient`
- обновляет рендер

---

#### Итог

- без `@property` → просто строка → нет анимации
- с `@property` → тип → есть интерполяция → есть анимация

👉 Важно:

`@property` не создаёт переменную.  
Он делает кастомное свойство полноценным CSS-свойством, которое браузер понимает.
