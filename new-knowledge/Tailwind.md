<a id="tailwind-q1"></a>
### 1. Что такое `@layer`?

`@layer` управляет порядком слоев в Tailwind.

Основные слои такие:

- `base` - базовые стили для тегов вроде `body`, `button`, `h1`
- `components` - переиспользуемые классы компонентов
- `utilities` - обычные utility-классы Tailwind вроде `p-4`, `flex`, `bg-black`

Порядок такой:

```txt
base -> components -> utilities
```

Почему это важно:
более поздний слой может переопределить более ранний.

В этом проекте:

- `@layer base` используется для общих правил
- `@layer components` используется для классов `.glass-wrap`, `.glass-surface`, `.glass-shadow`

Коротко:
`@layer` не создает стили сам.
Он помогает Tailwind и браузеру применять их в предсказуемом порядке.

<a id="tailwind-q2"></a>
### 2. В чем разница между `:root` и `@theme`?

Они похожи только на первый взгляд, потому что и там, и там можно хранить значения.

### `:root`

- это обычный CSS-селектор верхнего уровня
- в нем обычно хранят CSS-переменные
- потом эти переменные используются вручную через `var(--...)`

Пример:

```css
:root {
  --color-brand: #4f46e5;
}
```

### `@theme`

- это специальная директива Tailwind v4
- в ней задают дизайн-токены проекта
- Tailwind читает эти токены и создает на их основе utility-классы

Пример:

```css
@theme {
  --color-brand: #4f46e5;
}
```

После этого можно получить классы вроде `bg-brand` или `text-brand`.

Простая разница:

- `:root` - просто хранит CSS-переменные
- `@theme` - хранит значения и связывает их с системой Tailwind

<a id="tailwind-q3"></a>
### 3. Что делает `@theme` в Tailwind v4?

`@theme` описывает значения темы проекта.

Обычно туда выносят:

- цвета
- размеры текста
- радиусы
- отступы
- шрифты

В этом проекте в `@theme` лежат, например:

- `--color-background`
- `--color-primary`
- `--text-title`
- `--radius-glass-pill`

Что происходит дальше:

1. Ты объявляешь токены в `@theme`.
2. Tailwind читает их.
3. Tailwind создает utility-классы на основе этих значений.

Из-за этого в проекте можно писать классы вроде:

- `bg-background`
- `text-primary`
- `text-title`
- `rounded-glass-pill`

Коротко:
`@theme` связывает значения темы с классами Tailwind.

<a id="tailwind-q4"></a>
### 4. Что такое токен в контексте Tailwind и дизайн-системы?

Токен - это именованное значение с понятной ролью в интерфейсе.

Пример:

```css
--color-primary: #668eb9;
```

Это не просто цвет.
Это основной цвет интерфейса, у которого есть конкретная задача.

Почему токены удобны:

- у значения появляется понятное имя
- одно и то же значение можно использовать во многих местах
- если дизайн меняется, достаточно обновить токен в одном месте

### Чем токен отличается от обычной переменной?

Обычная переменная может быть технической и не объяснять свою роль:

```css
--temp-value: 17px;
```

По такому имени непонятно, зачем она нужна.

Токен обычно говорит о смысле:

```css
--color-primary: #668eb9;
--radius-glass-pill: 999vw;
```

Здесь уже ясно:

- `primary` - основной цвет
- `glass-pill` - радиус конкретного визуального элемента

Коротко:

- переменная хранит значение
- токен хранит значение со смыслом
